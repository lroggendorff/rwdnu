Responsive Web Design
And You

How did we get here?

The Web was intended for academic papers.

HTML provides semantic structure, CSS offers powerful aesthetic tools.

None of this was intended for pixel-perfection.

Web design came from print design where the work product has a defined edge.

Thus, a constant struggle:

  • table-based layouts
  • float and .clearfix
  • browser-specific hacks

The Web is not a book, is not a flyer, is not a poster, is not a painter’s canvas.

The Web is squishy.

Responsive: where we go from here

Three key ingredients:

  • A flexible, grid-based layout
  • Flexible images and media
  • Media queries
percentage widths, max-width: 100%

Media queries?

These guys:


@media only screen and (min-width: 480px) { ... }
@media only screen and (min-width: 600px) { ... }
@media only screen and (min-width: 768px) { ... }
@media only screen and (min-width: 992px) { ... }
@media only screen and (min-width: 1008px){ ... }
@media only screen and (min-width: 1382px){ ... }

Responsive: Not just for layout

To truly make content available anywhere and everywhere, don't forget bandwidth or performance.
